🔄 실시간 동기화 설정 (운영자 전용)

이 앱은 이미 Firebase에 연결되어 있습니다. 다른 Firebase 프로젝트로 바꾸고 싶을 때만 이 화면을 사용하세요. 직원분들은 이 화면을 볼 일이 없습니다.

📖 Firebase 설정 방법 (클릭해서 펼치기)
  1. https://console.firebase.google.com 접속 → 구글 로그인
  2. "프로젝트 추가" → 이름 입력 (예: daeryeong-pharm) → 만들기 (Analytics 끔)
  3. 좌측 메뉴 "빌드 → Realtime Database" 클릭 → "데이터베이스 만들기"
  4. 위치: asia-southeast1 (싱가포르) 선택 → 보안 규칙: "테스트 모드" 선택
  5. 좌측 톱니바퀴 ⚙️ → "프로젝트 설정" → 하단 "내 앱" 섹션에서 웹 아이콘 </> 클릭
  6. 앱 닉네임 입력 (예: 대령약국) → "앱 등록"
  7. 화면에 표시되는 const firebaseConfig = { ... }; 부분 전체를 복사
  8. 아래 칸에 붙여넣기 → 연결 버튼 클릭
⚠️ 테스트 모드는 30일 후 자동으로 잠깁니다. 약국 직원만 URL을 공유하시고, 30일 안에 보안 규칙을 강화하세요. (요청 시 도와드립니다)

📘 약품 사전 (주문 이력 기반 자동 구축)

최근 180일 주문 이력을 분석해 약품 계열·변형·규격·수량·비고 조합을 학습합니다.
각 주문 전송 시 실시간 반영되며, 모든 직원 자동완성에 최상위로 노출됩니다.

-
신규 주문은 자동 학습되므로 평소엔 불필요

🎓 수동 사전 학습 (모든 직원 입력 가능)

자주 주문하는 조합을 직접 입력해 사전에 추가합니다. 추가된 조합은 즉시 모든 직원의 자동완성에 노출됩니다.
형식: 의약품명 규격 갯수 비고   예시: 립스타플러스10/10 30정 1개 급송

📚 최근 추가된 학습 항목 (최대 30건) — 본인이 추가한 항목은 🗑로 삭제 가능

🔁 반품현황 대시보드 (반품 담당자: 김종희)

플랫폼별로 그룹화된 반품 의약품 현황. 모든 직원이 정보 조회 가능 · 추가·수정·상태변경은 김종희(반품 담당자)와 약국장만.

🔒 읽기 전용 — 반품 관리는 김종희(반품 담당자) 또는 백승준(약국장)만 가능합니다.

💰 의약품 단가 (처방조제 시스템 추출 데이터)

약국 처방조제 시스템에서 추출한 1년치 약품·1정 단가입니다. 주문 입력 시 자동 매칭되어 금액·일일 합계가 표시됩니다.
업로드 양식: 1번열=약품명 / 2번열=단가 (.xls/.xlsx) · 매월 갱신 권장
